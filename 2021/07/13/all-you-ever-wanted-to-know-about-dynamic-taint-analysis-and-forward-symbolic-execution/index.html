重要观点：

  • 动态分析的优点
    1. 结果准确

    2. 自动化过滤输入
    3. 测试用例生成

1.1. 动态污点分析

动态追踪source和sink

  • 两个挑战:overtainted和undertainting
  • 内容速览
    1. 如何执行动态符号执行?
      • 根据操作语义的规则
    2. 污点策略
      • introduction
      • transport
      • check
    3. 注意事项:哪些容易产生overtainted或undertainting?
      • decide when to remove a taint is more difficult than when to introduce a taint
      • undertainting ?
  • 重要观点: 不同目标应用的污点策略不同,但是大同小异

1.2. 符号执行

一个公式对应一条路径

  • 三个难题
    1. Symbolic Memory
      • 解决方案
        1. unsound assumptions
          将如下代码: 
          store(addr1, v)
z = load(addr2)
          作如下假设: 
          mem_addr1 = v
z = mem_addr2
          即忽略addr1与addr2是否为别名
        2. Let subsequent analysis steps deal with them.(滞后处理,比如之后再使用SMT判断二者是否存在别名关系)
        3. 别名分析
    2. System calls
      • 解决方案:副作用建模(规则?比如说使用上一次的执行结果)
    3. Path Selection
      将路径看成树,主要是要防止循环的出现(设置执行上限)
      • 解决方案
        1. DFS
        2. 实际执行
        3. 随机路径
        4. 启发式 ?
  • 其它
    • 符号跳转(跳转表)
      • 解决方案
        1. 实际执行
        2. SMT
        3. Static Analysis(作者认为效果最优)
    • 分析效果:三个指数
      1. 执行时间是程序分支数目的指数级
      2. 指数数量的formula
      3. 指数规模的formula size:例如x→s+s+s+s....+s
      • 解决方案:
        1. 硬件加速
        2. 简化约束
        3. 调整符号名→unique
        4. 分步求解
        5. weak precondition?
    • Mixed Execution:对于某些可信输入,不符号化
